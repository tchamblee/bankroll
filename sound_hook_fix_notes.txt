SOUND HOOK FIX NOTES
====================
Date: 2026-01-12

THE BUG:
--------
The PowerShell stop hook that plays complete.mp3 was failing with this error:

  "An expression was expected after '('."
  ...pointing to ".Play()" instead of "$mp.Play()"

The $mp variable was being stripped out, causing ".Play()" to be called
without an object reference.

ROOT CAUSE (CLAIMED):
---------------------
When the hook command runs, it passes through bash before reaching PowerShell.
Bash interprets $mp as an environment variable (which is empty/undefined),
so "$mp.Play()" becomes ".Play()".

THE FIX (APPLIED):
------------------
Escaped the dollar signs in ~/.claude/settings.json:
  Changed: $mp
  To: \$mp (which is \\$mp in JSON)

This should make bash pass the literal "$mp" through to PowerShell instead
of trying to expand it as a variable.

I stated this fix with confidence.
